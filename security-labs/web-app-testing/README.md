# Web Application Vulnerability Testing Workflows

> ⚠️ **Authorized / intentionally-vulnerable targets only** (e.g. DVWA, Juice Shop,
> your own apps).

## Objective
Document a repeatable methodology for assessing web applications — enumeration,
common vulnerability classes, and structured reporting.

## Workflow
1. **Recon &amp; mapping** — spider the app, identify entry points.
2. **Enumeration** — directories, parameters, technologies.
3. **Testing** — OWASP-style classes (injection, auth, access control, XSS, etc.).
4. **Reporting** — finding, impact, evidence, remediation.

## Tools Used
- Burp Suite
- Nikto / dirb / ffuf
- Browser dev tools
- Lab apps: DVWA, OWASP Juice Shop
